# Django course platform

Учебный проект на Django для управления онлайн-курсами, доступами пользователей и группами студентов.

## О проекте

Проект моделирует систему продажи и сопровождения курсов.  
У каждого курса есть автор, название, дата старта, цена и ограничения по количеству студентов в группе.  
Пользователь может получить доступ к курсу, после чего попадает в одну из групп этого курса.

## Что реализовано

### Продукт
Каждый продукт — это отдельный курс.  
У продукта есть:

- автор / преподаватель;
- название;
- дата и время старта;
- стоимость;
- минимальное количество студентов в группе;
- максимальное количество студентов в группе.

Это позволяет заранее задавать, сколько людей должно быть в каждой учебной группе и как курс будет наполняться.

### Доступ к курсу
Доступ пользователя к продукту хранится отдельно.  
Это нужно, чтобы понимать:

- кто оплатил или получил курс;
- какие курсы доступны конкретному пользователю;
- к каким урокам можно открыть доступ.

### Уроки
Каждый урок принадлежит только одному продукту.  
В уроке хранится:

- название;
- ссылка на видео.

Таким образом, уроки строго привязаны к конкретному курсу и не смешиваются между продуктами.

### Группы
Для каждого продукта могут быть несколько групп студентов.  
В группе хранится:

- название группы;
- продукт, к которому она относится;
- список учеников, которые в ней состоят.

Группы нужны для распределения студентов внутри курса и для контроля загрузки.

## Как работает распределение

Когда пользователь получает доступ к продукту, он автоматически распределяется в группу этого продукта.

Логика распределения учитывает ограничения курса:

- если в группе еще есть место, пользователь добавляется туда;
- если группа заполнена, создается или выбирается следующая;
- если курс еще не начался, группы можно перераспределить более равномерно;
- при необходимости можно выравнивать количество участников так, чтобы разница между группами была минимальной.

### Минимум и максимум
Для каждого курса задаются границы наполнения группы:

- **минимум** — меньше этого числа группа считается неполной;
- **максимум** — больше этого числа добавлять пользователей нельзя.

Это помогает держать группы в рабочем размере и не перегружать преподавателя.

### Распределение по группам
Пользователь попадает в ту группу, где еще есть место.  
Если все группы заполнены, можно:
- создать новую группу;
- пересобрать группы, если курс еще не начался;
- выровнять нагрузку между группами.

То есть курс не просто выдает доступ, а сразу организует студента в учебную структуру.

## Публичная часть сайта

Для сайта также сделаны основные страницы:

- главная;
- список курсов;
- о нас;
- контакты;
- профиль пользователя;
- вход;
- регистрация;
- выход.

Общий шаблон `base.html` объединяет навигацию и делает сайт удобным для переходов между страницами.

## API

В проекте также есть API:

- список доступных продуктов для покупки;
- список уроков конкретного продукта, если у пользователя есть доступ.

Это удобно для подключения фронтенда, мобильного приложения или отдельного кабинета.
